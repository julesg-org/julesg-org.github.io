Setting Up a Quarto Website for Your GitHub Organisation

quarto
github-pages
tutorial

A step-by-step guide to creating a Quarto-powered static site for a GitHub organisation, including automatic deployment with GitHub Actions.

Author

Jules G Org

Published

February 1, 2024

Introduction

Quarto makes it straightforward to publish a polished organisation website directly from a GitHub repository. This post walks through the setup used for this very site.

Prerequisites

  • A GitHub account / organisation
  • Quarto CLI installed locally
  • Basic familiarity with Markdown

Step 1 — Create the Repository

On GitHub, create a repository named <org>.github.io (for organisations) or <username>.github.io (for personal sites). This special naming convention tells GitHub Pages to serve the repo at https://<org>.github.io.

For example:

Organisation: julesg-org
Repository:   julesg-org.github.io   ← served at https://julesg-org.github.io

Step 2 — Initialise a Quarto Website

# In your cloned repository folder:
quarto create project website .

This creates the minimal _quarto.yml and index.qmd scaffold.

Step 3 — Customise _quarto.yml

Open _quarto.yml and set at minimum:

project:
  type: website

website:
  title: "Your Org Name"
  site-url: https://your-org.github.io
  navbar:
    left:
      - text: "Home"
        href: index.qmd
      - text: "About"
        href: about.qmd

Step 4 — Add GitHub Actions

Create .github/workflows/deploy.yml to automate builds on push:

name: Deploy
on:
  push:
    branches: [main]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: quarto-dev/quarto-actions/setup@v2
      - run: quarto render
      - uses: peaceiris/actions-gh-pages@v4
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./_site

Step 5 — Enable GitHub Pages

In your repository: Settings → Pages → Source → gh-pages branch.

Push to main and the site deploys automatically! 🎉
